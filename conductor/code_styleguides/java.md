# Java Code Style Guide

## General
- Follow the [Google Java Style Guide](https://google.github.io/styleguide/javaguide.html) for formatting and conventions.
- Use **CamelCase** for classes and interfaces (e.g., `GameController`).
- Use **camelCase** for variables and methods (e.g., `calculateScore`).
- Use **UPPER_SNAKE_CASE** for constants (e.g., `MAX_RETRIES`).

## File Structure
- One top-level class per file.
- Order class members: constants, fields, constructors, methods.

## Formatting
- Use 4 spaces for indentation.
- Limit line length to 100 or 120 characters.
- Use K&R style braces (opening brace on the same line).

## Comments & Javadoc
- Use Javadoc for all public classes and public/protected methods.
- Comments should explain *why*, not *what*.
